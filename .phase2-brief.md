## Workstream 1 — `dashboard/src/components/ui/` primitives

Create ~10 token-based primitives. Study the existing patterns FIRST (Badge.tsx, DeleteConfirmModal.tsx, the toast system, any focus-trap util) and CONSOLIDATE — extend or wrap existing implementations, never create a parallel duplicate.

- `Button.tsx` — variants primary/secondary/ghost/danger; sizes sm/md; `loading` prop (spinner, disabled); focus ring `box-shadow: 0 0 0 3px var(--brand-glow)`; primary gets the resting brand glow from the hero Resume button.
- `Card.tsx` — `level` prop: 'surface' (default) | 'elevated'; hairline border, `--radius-lg`; optional `toolEdge` prop taking a CSS color for the Phase-1b 3px left edge.
- `Input.tsx`, `Textarea.tsx`, `Select.tsx` — unified field styling: `--bg-sunken` background, hairline border, focus border `--brand` + glow ring, `error` prop (danger border + message slot).
- `Dialog.tsx` — `--overlay` surface, backdrop `rgba(0,0,0,0.5)`, focus-trapped (REUSE the existing focus-trap logic), Esc-close, ARIA role=dialog + aria-modal + labelled-by.
- `Tabs.tsx` — pill style matching the Phase 1b nav (surface pill + hairline border on active).
- `Table.tsx` — dense rows, hairline row dividers, 13px cells, header 11px uppercase `text-tertiary`.
- `Toast` — restyle the EXISTING toast system onto tokens (elevated surface, semantic left edge by type); do not build a second toast system.
- `Dropdown.tsx` — `--overlay` surface menu, full keyboard nav (arrows/Enter/Esc), ARIA menu roles.
- `Tooltip.tsx` — 400ms delay, `--overlay`, 12px.
- `Kbd.tsx` — generalize the ⌘K chip (mono, sunken bg, hairline border).

Barrel export `ui/index.ts`. Add basic tests per primitive (render, variant classes, a11y attributes, keyboard behavior for Dialog/Dropdown).
