## Token spec (dark = default)

1. **Themes flip:** `:root, [data-theme="dark"]` becomes the dark block; `[data-theme="light"]` the override. ThemeToggle default (no stored preference) = dark; stored preferences still respected.

2. **Dark neutral ladder — OKLCH, slightly cool (hue ~255, chroma 0.010–0.015), perceptually even ~3–4% lightness steps:**
   - `--bg-sunken` ≈ L 14% (wells, code blocks — NEW token)
   - `--bg-primary` (canvas) ≈ L 16%
   - `--bg-secondary` (sidebar/nav) ≈ L 18%
   - `--surface` (cards) ≈ L 21%
   - `--bg-elevated` (hover cards, popovers) ≈ L 24%
   - `--overlay` (modals, dropdowns) ≈ L 27% (NEW token)
   - `--border` hairline ≈ L 30%; `--border-strong` ≈ L 40%
   - text: `--text-primary` L 93%, `--text-secondary` L 72%, `--text-tertiary` L 55%; verify ≥ 4.5:1 contrast for primary AND secondary text on `--surface`
   - THE POINT: surface ≠ elevated ≠ canvas. The current dark theme's depth collapse (`--bg-elevated` == `--surface` == #1c2128) is the headline bug you are fixing.
   - Set `--bg-tertiary` to a sensible rung (≈ L 18–19%) so legacy users of it still look right.

3. **Light theme:** same ladder inverted (canvas ≈ L 97%, surface = white, sunken ≈ L 94%, etc.) on OKLCH steps — keep the current light feel. Light keeps soft shadows; dark relies on surface steps + hairline borders (keep only a faint ambient shadow at overlay level in dark).

4. **Brand + semantic (unchanged values):** `--brand` #4f9cf7 (dark) / #1a73e8 (light), `--accent` #3ddc84, danger/warning/info, ALL `--tool-*` colors. ADD `--brand-glow: color-mix(in srgb, var(--brand) 25%, transparent)` and use it for focus rings.

5. **Typography:** body + display → **Inter** variable (Google Fonts import, weights 400–800). JetBrains Mono stays. Remove Plus Jakarta Sans entirely from the dashboard. Add a `.text-mono-chip` utility (font-mono, 0.75rem, color tertiary, letter-spacing 0.02em, slight bg + radius — for session IDs / etags / tool names).

6. **Keep INTACT:** the whole `@theme` legacy mapping block (`bg-bg-primary`, `text-text-primary`, etc. must keep resolving — map any new tokens in additively), role colors, severity colors, radius scale, skeleton/page-enter/tab animations, scrollbar styling.
