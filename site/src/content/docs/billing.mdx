SessionFS billing is managed through SessionFS Cloud. Self-hosted customers do not run billing infrastructure.

## Start with cloud

Create an account at [app.sessionfs.dev](https://app.sessionfs.dev/login?mode=signup). The dashboard creates your API key, walks you through setup, and lets you upgrade when you need cloud sync, shared knowledge, or team coordination.

## Plans

| Tier | Price | Best for | Key features |
|------|-------|----------|--------------|
| **Free** | $0 | Local development | Capture, resume, local search, local MCP |
| **Starter** | $4.99/mo | Solo cloud sync | Cloud sync, dashboard, bookmarks, manual Judge, deterministic summaries |
| **Pro** | $14.99/mo | Solo agent memory | Starter + autosync, remote MCP, project context, knowledge base, rules portability, DLP, agent personas |
| **Team** | $14.99/user/mo | Shared agent coordination | Pro + org management, shared storage pool, tickets, team handoff, shared personas |
| **Enterprise** | Custom | Governed AI agent operations | Team + self-hosted or single-tenant deployment, HIPAA DLP, SSO, audit retention, AgentRun CI enforcement, cloud agent support |

See [Pricing](/pricing/) for the current public plan summary.

## Cloud storage

| Tier | Cloud storage |
|------|---------------|
| Free | 50 MB sync limit |
| Starter | 500 MB |
| Pro | 500 MB |
| Team | 1 GB per seat, pooled |
| Enterprise | Custom |

Storage is checked on every sync upload. If you exceed your limit, the server rejects the upload with a clear error and your current tier's limit.

## Manage a subscription

Billing changes happen in the dashboard:

1. Open [app.sessionfs.dev](https://app.sessionfs.dev/login).
2. Go to **Settings > Billing**.
3. Choose a plan or open the subscription page for invoices, payment methods, upgrades, downgrades, and cancellation.

Team billing is owned by organization admins. Members inherit the organization's tier and storage pool.

## Self-hosted and enterprise

Self-hosted customers do not configure billing infrastructure in their cluster. Enterprise billing, renewals, and licensed features are handled through your SessionFS agreement.

For self-hosted deployments, the runtime receives its allowed tier from the license system. The application should be deployed with the license material provided by SessionFS, not with any customer-operated checkout, subscription, or payment-webhook configuration.

Talk to [Enterprise](/enterprise/) if you need self-hosted, single-tenant, HIPAA-oriented, or custom-retention deployment.
